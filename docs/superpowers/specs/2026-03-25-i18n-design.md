# Internationalization (i18n) Design Specification

**Date:** 2026-03-25
**Scope:** Multi-language support for Diabetes Risk Calculator (DE, EN, FR, ES)
**Architecture:** Vanilla JS, no bundler, window.DRC namespace

---

## 1. Overview

Implement a persistent language selection system with a dropdown button in the top navigation. The system must translate the entire interface including all menus, modals, labels, and dynamic content.

### Supported Languages
| Code | Language | Flag |
|------|----------|------|
| en | English | 🇺🇸 |
| de | German | 🇩🇪 |
| fr | French | 🇫🇷 |
| es | Spanish | 🇪🇸 |

### Storage
- **localStorage key:** `drc-language`
- **Default:** `en`
- **Persistent:** Yes (survives page reloads)

---

## 2. Architecture

### Module Structure

```
js/i18n/
├── i18n-service.js          # Core translation service
├── i18n-ui.js               # Language selector button & dropdown
└── translations/
    ├── en.json               # English (source of truth)
    ├── de.json               # German
    ├── fr.json               # French
    └── es.json               # Spanish
```

### Core Service API (I18nService)

```javascript
// Initialization
DRC.I18n.init() -> Promise<void>

// Get/set current language
DRC.I18n.getCurrentLang() -> string
DRC.I18n.setLanguage(langCode) -> Promise<void>

// Translation
DRC.I18n.t(key, fallback?) -> string
DRC.I18n.translateDOM() -> void

// Events
DRC.I18n.on('language:changed', callback)
```

### HTML Integration

Static text elements use `data-i18n` attributes:
```html
<button data-i18n="nav.reset">Reset</button>
<span data-i18n="factors.age">Age</span>
```

Dynamic content uses JavaScript API:
```javascript
const title = DRC.I18n.t('hero.title');
```

## 3. UI Design

### Language Selector Button

**Location:** Top navigation, right of Patient Menu button

**Structure:**
```html
<button class="btn-language" aria-label="Select language" aria-expanded="false">
    <span class="lang-flag" aria-hidden="true">🇺🇸</span>
    <span class="lang-code">EN</span>
    <i data-lucide="chevron-down" class="lucide-icon"></i>
</button>

<div class="lang-dropdown" id="langDropdown" hidden>
    <button class="lang-option" data-lang="en">
        <span class="lang-flag">🇺🇸</span>
        <span class="lang-name">English</span>
    </button>
    <button class="lang-option" data-lang="de">
        <span class="lang-flag">🇩🇪</span>
        <span class="lang-name">Deutsch</span>
    </button>
    <button class="lang-option" data-lang="fr">
        <span class="lang-flag">🇫🇷</span>
        <span class="lang-name">Français</span>
    </button>
    <button class="lang-option" data-lang="es">
        <span class="lang-flag">🇪🇸</span>
        <span class="lang-name">Español</span>
    </button>
</div>
```

**Styling Requirements:**
- Match existing `btn-patient-menu` styling
- Height: 32px, border-radius: 20px
- Background: transparent → rgba(0,0,0,0.03) on hover
- Dropdown: card shadow, 8px border-radius, backdrop blur
- Active language: highlighted background

### Dropdown Behavior

| Action | Result |
|--------|--------|
| Click button | Toggle dropdown |
| Click outside | Close dropdown |
| Press ESC | Close dropdown |
| Select language | Close dropdown, apply translation |

## 5. Integration Points

### Files to Modify

| File | Changes |
|------|---------|
| `index.html` | Add language button, data-i18n attributes, include i18n scripts |
| `style.css` | Language selector styles, dropdown animation |
| `js/config.js` | Add I18N_CONFIG constant |
| `js/ui-controller.js` | Listen for language:changed event |
| `js/app.js` | Initialize I18n service on startup |
| `js/patient-manager.js` | Translate UI labels |

```

2.  xlsx.full.min.js
3.  config.js
4.  i18n/translations/en.json (inline as JS object for fallback)
5.  i18n/i18n-service.js      ← NEW
6.  i18n/i18n-ui.js           ← NEW
7.  conversion-service.js
8.  risk-model.js
9.  ui-helpers.js
10. ui-controller.js
11. radar-chart.js
12. timeline-chart.js
13. app.js                     ← Init I18n here
14. patient-manager.js
15. treatment-simulator.js
16. dark-mode.js
17. main.js
```

## 6. Error Handling

| Scenario | Behavior |
|----------|----------|
| Translation file 404 | Fallback to English, log warning |
| Missing translation key | Return key name (e.g., "nav.missing") |
| Invalid language code | Default to English |
| localStorage disabled | Use session-only (no persistence) |

## 7. Testing Checklist

- [ ] Language persists after page reload
- [ ] All static text translated via data-i18n
- [ ] All dynamic content translated via JS API
- [ ] Dropdown opens/closes correctly
- [ ] Active language highlighted in dropdown
- [ ] Flag updates when language changes
- [ ] Works with existing unit toggle
- [ ] Works with existing dark mode
- [ ] No console errors on language switch
- [ ] Falls back to English on missing translation

---

## 8. Performance Considerations

- Lazy load translation files (fetch on demand)
- Cache loaded translations in memory
- Batch DOM updates on language switch
- Use requestAnimationFrame for DOM traversal

---

## 9. Accessibility

- `aria-expanded` on language button
- `aria-label` on all language options
- Keyboard navigation (arrow keys, Enter, Escape)
- Focus management on dropdown open/close
- Screen reader announcements on language change
